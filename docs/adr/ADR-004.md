# ADR-004: Manual payment proof review only for personal QR codes

- Status: accepted
- Date: 2026-06-25

## Context
The project forbids personal-code monitoring, cookie/notification reading and "免签" automation. Payment proof handling must avoid storing raw tokens, cookies or production personal data in Git/evidence.

## Decision
Support manual payment proof review only for personal QR code flows. Do not automate personal-code monitoring or treat screenshots/messages as automated payment confirmation.

## Consequences
- Manual review must produce auditable status changes without weakening entitlement checks.
- Payment proof uploads must be validated, access-controlled and redacted in evidence.
- Future payment automation requires a separate approved ADR and compliance review.

## Verification
- Repository rules and task evidence contain no raw payment credentials, cookies or personal-code automation.
- This ADR is policy/compliance-driven and does not require EDD runtime hooks to accept the prohibition.
