# ADR-005: Private object storage with one-time download tokens

- Status: accepted
- Date: 2026-06-25

## Context
SR-003 provides MinIO as the local object store. SR-006 verified private object behavior, 120-second presigned GET URLs, Range support and expiry behavior using disposable objects.

## Decision
Use private object storage and issue short-lived signed URLs from the backend. Object keys and entitlement/download tokens remain server-authoritative; public bucket access is not allowed.

## Consequences
- Downloads must go through a backend service that checks entitlement/quota before issuing a signed URL.
- Signed URLs should be short lived; 120 seconds is accepted for W1 design.
- Range requests are compatible with signed GET URLs and can be used for resumable downloads.

## Verification
- Direct unauthenticated GET to a private MinIO object returned 403.
- Presigned GET in the Compose network returned the object content.
- Range request returned HTTP 206 Partial Content.
- A 1-second presigned URL returned 403 after expiry.
