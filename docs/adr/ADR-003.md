# ADR-003: Self-build entitlement engine instead of EDD All Access

- Status: proposed-blocked
- Date: 2026-06-25

## Context
The platform requires entitlement rules beyond basic EDD order storage. Runtime EDD order/refund validation is blocked, so the adapter boundary is not yet proven against live EDD data.

## Decision
Keep the proposed direction: self-build the entitlement engine instead of relying on EDD All Access. Acceptance is blocked until EDD runtime semantics are proven and the entitlement inputs are confirmed.

## Consequences
- Entitlement code must consume normalized data from `EddOrderAdapter`, not query EDD internals directly.
- No paid EDD entitlement plugin is approved by this ADR.

## Verification
- MariaDB transaction/unique-key/deadlock behavior was verified for quota-style consistency needs.
- EDD runtime order/refund inputs remain unverified.
