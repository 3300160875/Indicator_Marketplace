# ADR-001: Use Roots Bedrock as repository foundation

- Status: accepted
- Date: 2026-06-25

## Context
SR-002 initialized Roots Bedrock 1.31.0 with Composer-managed WordPress 7.0 and EDD 3.6.9. SR-003 verified the local Docker environment can run the Bedrock document root through Nginx and PHP-FPM without committing generated WordPress, EDD or vendor artifacts.

## Decision
Use Roots Bedrock as the repository foundation and keep WordPress core/plugins/themes Composer-managed and ignored in Git.

## Consequences
- Application-owned code must live outside WordPress/EDD core directories.
- Environment-specific secrets live in ignored `.env`, with `.env.example` containing development placeholders only.
- Local and CI tooling should install dependencies from lockfiles rather than committing generated dependency trees.

## Verification
- `composer validate --strict` passed.
- `make bootstrap` and `make test-smoke` passed in SR-004 after wrapping SR-003 Docker scripts.
- `git ls-tree` checks during SR-003 review found no committed `vendor/`, `web/wp/`, EDD plugin or default theme directories.
