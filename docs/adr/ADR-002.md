# ADR-002: Use EDD Core as order and customer source of truth

- Status: proposed-blocked
- Date: 2026-06-25

## Context
Composer locks EDD 3.6.9, and static inspection found public hooks around order completion and refunds. Runtime validation is not complete because the local database does not have WordPress installed, EDD is not activated, WP-CLI is unavailable, and SR-006 allowed paths do not permit committing a disposable spike plugin or runner.

## Decision
Keep the intended direction of using EDD Core as order/customer source of truth, but do not mark this ADR accepted until runtime spike evidence proves order creation, completion hooks, order item IDs and refund data.

## Consequences
- SR-031 or a resumed SR-006 must install WordPress in a disposable database, activate EDD, create throwaway order/refund data, and observe hooks before implementing `EddOrderAdapter`.
- Business code must not depend only on static grep results.

## Verification
- Static evidence: EDD 3.6.9 is locked and installed by Composer.
- Static evidence: `edd_complete_purchase` is hooked to `edd_update_payment_status` in EDD payment actions.
- Blocker: no runtime order/refund hook evidence yet.
