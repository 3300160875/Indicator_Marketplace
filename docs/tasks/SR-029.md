## Reference implementation sequence

1. 读取 `SR-029` 的依赖任务证据、相关 ADR/OpenAPI/Schema，并确认允许路径未被锁定。
2. 在 `docs/evidence/SR-029/` 建立证据目录，先写失败测试或可复现检查。
3. 文件先进入 quarantine 前缀，不写公开媒体库。
4. 服务端嗅探 MIME、限制文件数/嵌套/解压体积/压缩比并计算 SHA-256。
5. 调用扫描器并持久化 scan_status/result；失败不得激活。
6. 审核通过后原子切换 current 版本并移动/复制到正式前缀。
7. 执行下列命令；任何失败不得通过删测试或降低校验绕过。
8. 更新任务卡的契约/迁移影响、测试结果、回滚和下一安全任务；提交 REVIEW。

> Agent 可以细化步骤，但不得删除安全、测试、迁移和证据步骤；若实际版本行为与文档不同，停止并标记 BLOCKED。
