## Reference implementation sequence

1. 读取 `SR-053` 的依赖任务证据、相关 ADR/OpenAPI/Schema，并确认允许路径未被锁定。
2. 在 `docs/evidence/SR-053/` 建立证据目录，先写失败测试或可复现检查。
3. 生成 32 字节 CSPRNG token，外部 Base64URL；数据库只存带应用密钥的 HMAC。
4. token 绑定用户/资源/版本/权益/配额预占，TTL 默认 120 秒。
5. 验证 raw token 不进入日志、审计、异常或 URL 查询分析。
6. 执行下列命令；任何失败不得通过删测试或降低校验绕过。
7. 更新任务卡的契约/迁移影响、测试结果、回滚和下一安全任务；提交 REVIEW。

> Agent 可以细化步骤，但不得删除安全、测试、迁移和证据步骤；若实际版本行为与文档不同，停止并标记 BLOCKED。
