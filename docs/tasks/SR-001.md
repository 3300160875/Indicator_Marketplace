## Reference implementation sequence

1. 读取 `SR-001` 的依赖任务证据、相关 ADR/OpenAPI/Schema，并确认允许路径未被锁定。
2. 在 `docs/evidence/SR-001/` 建立证据目录，先写失败测试或可复现检查。
3. 确认受控基线与 SHA-256，并登记优先级。
4. 建立 ADR、任务、状态、风险与证据模板。
5. 运行文档一致性检查并只把 SR-001 置为 READY。
6. 执行下列命令；任何失败不得通过删测试或降低校验绕过。
7. 更新任务卡的契约/迁移影响、测试结果、回滚和下一安全任务；提交 REVIEW。

> Agent 可以细化步骤，但不得删除安全、测试、迁移和证据步骤；若实际版本行为与文档不同，停止并标记 BLOCKED。
