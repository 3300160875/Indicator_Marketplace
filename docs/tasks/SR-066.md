## Contract / data / migration impact — completion required

- API/OpenAPI: no production API change; adds local-only `stock-resource-e2e/v1` runtime routes gated by `SR_E2E_ENABLED=1` and local/development environment.
- Database/migration: no migration; E2E stores temporary per-run WordPress options with `sr_e2e_p0_run_<md5>` keys.
- Events/Hooks: E2E runtime loader conditionally requires `tests/e2e/wp/e2e-runtime.php`; the runtime uses WordPress REST and EDD order/refund APIs in local/CI.
- Configuration/Feature Flags: adds `SR_E2E_ENABLED` and `SR_E2E_KEY` container env vars; runtime is disabled unless both E2E enabled and a non-empty key are present.
- Cache/invalidation: E2E bootstrap deletes temporary `sr_e2e_p0_run_%` options.
- Backward compatibility: production/default local runtime unchanged when `SR_E2E_ENABLED` is not `1`.
- Observability/audit: Playwright report/results capture failures; CI uploads artifact `sr-066-playwright-evidence`.
