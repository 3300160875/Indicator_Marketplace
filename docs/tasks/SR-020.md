- Code rollback: revert SR-020 commit/PR.
- Feature flag / kill switch: do not invoke `bin/seed-resources`.
- Data forward-fix or restore: delete the caller-provided seed state file; no runtime database is touched.
- Reconciliation command: rerun `bin/seed-resources --fixtures tests/fixtures/resources --state <state-file>`.
- User-facing degradation: none; fixtures are test/dev data only.
