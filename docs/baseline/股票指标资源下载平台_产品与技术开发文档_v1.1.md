# 股票指标资源下载平台：产品需求、技术架构与研发实施文档

**架构：** WordPress + Easy Digital Downloads Core + 自研 `sr-entitlements` 权益引擎  
**版本：** V1.1  
**日期：** 2026-06-24  

> 本文件是 DOCX 正式文档的便携摘要。完整的页面级功能、数据字段、权限矩阵、并发规则、测试用例、研发排期与上线清单以 DOCX 为准。

## 1. 本次架构决策

不采购 EDD All Access。EDD Core 只负责商品、价格、订单、客户、优惠码和订单状态；自研 `sr-entitlements` 负责 VIP 套餐、权益实例、适用范围、排除项、下载配额、续期、撤权、内容限制和会员中心。

所有资源详情、购买按钮、隐藏内容和下载接口统一调用 `EntitlementService`，主题不得直接查询订单表或用 CSS/JavaScript 模拟权限。

## 2. 首期插件边界

- `sr-core`：股票指标资源字段、分类、版本、公共 API、SEO 与领域事件。
- `sr-entitlements`：套餐、权益、范围、配额、续期/撤权、内容限制、会员中心。
- `sr-payment-gateways`：个人码付款凭证、人工审核及未来正式支付适配。
- `sr-private-downloads`：私有对象存储、一次性令牌、短时签名链接、下载事件与风控。
- `sr-admin-ops`：资源审核、版权、工单、报表、导入和审计工作台。
- `stock-resource-theme`：服务端渲染前台主题，只展示公共服务返回的状态。

## 3. 自研权益核心设计

VIP 套餐仍作为特殊 EDD Download 销售，并以 `_sr_product_type=membership_plan` 标记。订单完成后按订单项快照创建权益；套餐后续修改不追溯改变存量会员。

核心数据表：

- `sr_entitlements`：权益来源、有效期、范围快照、配额快照、状态与撤销信息；`source_order_item_id` 唯一，保证重复审核或重复回调不会重复授权。
- `sr_entitlement_counters`：按权益与日/周/月/总周期保存已用、预占和限额快照，支持并发原子扣减。
- `sr_download_events`：保存请求 ID、资源、版本、授权来源、计数结果和下载结果，用于审计、失败补偿和对账。

鉴权优先级：免费资源 → 单品已购 → 管理员授权 → 有效 VIP → 拒绝。返回完整 `AccessDecision`，包含 `allowed`、`source`、`reason_code`、`entitlement_id`、是否扣配额、剩余次数和到期时间。

## 4. 首期支付

普通个人收款码不具备标准网站支付回调能力。首期采用：

订单创建 → 用户查看付款说明 → 上传付款凭证 → 财务核对真实账单 → 批准或驳回 → EDD 订单完成 → 幂等发放单品或 VIP 权益。

不开发通知监听、Cookie 读取、挂机猜单或来源不明的免签接口。付款功能生产默认关闭，只有 Gate 0 通过后才能启用。

## 5. P0 模块

1. 首页、分类、标签、筛选、搜索、资源详情、教程和 VIP 页面。
2. 注册登录、订单、会员中心、下载中心、收藏和工单。
3. 资源属性、版本、文件、截图、版权证据、技术与合规审核。
4. EDD 单品订单、套餐订单、订单快照与人工付款审核。
5. 自研 VIP 套餐、权益生命周期、范围、排除项、每日配额和内容限制。
6. 私有存储、一次性令牌、防直链、下载日志、失败补偿和异常控制。
7. 运营后台、财务审核、会员工作台、客服、报表、审计和配置。
8. CI/CD、缓存、监控、备份、恢复、安全、性能和升级回归。

## 7. 权益专项验收

- 同一订单重复完成或审核 10 次，只创建或延长一次权益。
- 单品购买不消耗 VIP 配额；退款后立即阻止新下载令牌。
- 过期、排除资源、配额耗尽、多套餐并存和人工授权结果唯一且可解释。
- 剩余 1 次时 50–100 并发请求最多一个成功预占。
- 下载失败、签名失败或存储故障不会错误计为成功下载。
- 修改前端、直调 REST API 或复制对象存储地址都不能绕过鉴权。
- WordPress、EDD 和自研权益插件升级后，核心链路回归全部通过。

## 8. Gate 0

上线付费前必须确认经营主体与收款方式、证券投资咨询边界、内容版权、消费者规则、隐私和网站部署/备案等要求。未通过时只上线免费内容，付款功能保持关闭。
