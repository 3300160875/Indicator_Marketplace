
# SR-006 技术 Spike 清单

Spike 必须在一次性分支/测试环境运行，不把试验代码直接并入业务模块。

## EDD
- [ ] 建单、订单项 ID、客户/用户映射；
- [ ] `edd_complete_purchase` 同步时点和重复调用；
- [ ] 后台人工完成订单时 Hook 行为；
- [ ] 订单项退款、整单退款、取消时的数据和 Hook；
- [ ] 可用公开 API，禁止依赖私有实现；
- [ ] WordPress 7.0 + PHP 8.3 兼容和日志。

## 数据库
- [ ] MariaDB 10.11 事务、唯一键冲突和 `SELECT ... FOR UPDATE`；
- [ ] 两连接并发配额预占；
- [ ] 死锁重试策略；
- [ ] dbDelta 局限和自研迁移工具可行性。

## 存储
- [ ] MinIO 私有对象上传、HEAD、删除；
- [ ] 120 秒预签名 URL；
- [ ] 令牌重放和 Range 请求；
- [ ] 失败时配额释放。

## 输出
每项保存：代码片段、命令、版本、日志、结论、风险、推荐接口；最终完成 ADR-001～006。
